# Vertragsdokumente — Verzeichnis

Übersicht über alle vertraglich relevanten Dokumente der Axano GmbH im
Zusammenhang mit Axano LeadFlow. **Die Dokumente selbst liegen im internen
Drive (Notion / Google Drive)**, nicht in diesem Repository — siehe Abschnitt
„Repo-Sichtbarkeit" in [`../DSGVO/README.md`](../DSGVO/README.md).

## Mustertext-AV-Vertrag (für Axano-Kunden)

| Feld | Wert |
|---|---|
| Zweck | Vertrag, mit dem Axano (Auftragsverarbeiter) und Kunden (Verantwortliche) die Lead-Daten-Verarbeitung gemäß DSGVO Art. 28 regeln |
| Anwendung | Bei jedem neuen Kunden vor Live-Schaltung der ersten Kampagne — Mustertext kopieren, Kundennamen einsetzen, beidseitig digital signieren |
| Speicherort | Internes Drive (Notion / Google Drive) |
| Status | **Final — anwaltlich freigegeben am 29.04.2026** |
| Letzte Aktualisierung | 29.04.2026 |
| Pflege | Bei Subprocessor-Änderungen oder Service-Erweiterungen aktualisieren und Versionsnummer erhöhen |

### Workflow für neue Kunden

1. Mustertext aus internem Drive kopieren
2. Kundennamen, Adresse, Vertragsdatum einsetzen
3. An Kunden zur Prüfung schicken
4. Beidseitig digital signieren (DocuSign / qualifizierte elektronische Signatur)
5. Signierte Version im internen Drive ablegen unter `Recht / AV-Vertraege / [Kundenname]-[Datum].pdf`
6. **NIEMALS** signierte Verträge ins Repo committen

## Signierte AV-Verträge mit Drittanbietern

Status pro Anbieter siehe [`../DSGVO/AV-VERZEICHNIS.md`](../DSGVO/AV-VERZEICHNIS.md) (sobald angelegt).

Die signierten Verträge / DPAs liegen im internen Drive unter
`Recht / AV-Vertraege / Drittanbieter / [Anbietername].pdf`.

## Was NICHT hier rein darf

- Mustertext-Vertragstext selbst (liegt intern)
- Signierte Verträge (Kundenname + Vertragsdatum sind sensibel)
- Vertragsentwürfe vor Anwaltsfreigabe
- Honorar- oder Preislisten
